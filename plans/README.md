## Findings considered and rejected

- **`protocol.ts:8` non-null assertion** (`Object.values(cfg.presets)[0]!`): not worth fixing because `validatePresets` in `config-validate.ts:75-85` enforces `presets` has ≥1 entry at load time. A fail-fast `throw` would be worse UX than the assertion.
- **`config-validate.ts:49` cast** (`raw as unknown as RouterConfig`): not worth fixing because every sub-validator (`validateRootFields`, `validatePresets`, etc.) runs before the cast; adding a runtime schema library (zod/io-ts) is scope growth beyond the audit's intent.
- **`delegate.ts:329` `void err` discards retryable error context**: LOW impact — the error is already logged via `logEvent.routing.retryable` at lines 320-324 with `classified.reason`. The raw stack is not emitted, but the classified reason is the operator-actionable signal. Deferred; revisit if operators report insufficient retry diagnostics.
- **`config-store.ts:200` refresh logging gated behind `MODEL_ROUTER_LOG=1`**: LOW impact — the public methods (`isStale`, `loadedAtMs`) provide observability without the log line. The env gate is intentional (avoids log noise by default). Not a finding.
- **`delegate.ts` 462-line function size**: MEDIUM impact but M effort and M risk — splitting `executeDelegate` into ~4 focused modules (config-resolve, session-run, gate-eval, cleanup) is a worthwhile refactor but large enough to warrant its own plan after 001-006 land and the test coverage from 002 is in place. Deferred to a future audit cycle.

### Audit cycle 2 (2026-07-01, commit `0400624`)

- **Config cache TTL bypass on the hot path** (`src/plugin/hooks.ts:361`): `handleSystemTransform` calls `getFreshConfig()` on every orchestrator message, forcing a disk read that bypasses the 5-minute TTL cache. Switching to `getConfig()` is an S-effort fix with LOW risk. Not planned in this cycle because it is lower leverage than the baseline, reasoning, and CI fixes; schedule it in the next cycle after 011 lands.
- **Prototype pollution in `deepMergeConfig`** (`src/router/config-loader.ts:208-213`): real unsafe pattern but LOW practical exploitability (config layers are trusted operator files). S-effort hardening (skip `__proto__`/`constructor` keys). Deferred; schedule alongside other config-loader hardening.
- **Duplicated verification event emission and cleanup** (`src/plugin/delegate.ts:401-417` vs `src/verify/dispatch.ts:375-406`): M-effort refactor to extract shared helpers. Lower priority than the baseline fix; schedule after 011 lands so the refactor can rely on the integration tests it restores.
- **README Node.js version mismatch** (`README.md:737` says "18+", `package.json:22` requires ">=20.0.0"): trivial S-effort doc fix. Not worth a standalone plan; fix opportunistically when touching README.
- **`.gitignore` missing `.env*` pattern** (`.gitignore:1-8`): preventive hygiene, S-effort. Not worth a standalone plan; fix opportunistically.
- **`PluginContext` god object** (`src/plugin/context.ts:72-139`, 15+ fields): justified by the plugin SDK's single-context design. Splitting into sub-contexts would require refactoring every hook adapter with MED risk for no behavioral gain. Not worth doing unless the SDK adds per-hook context scoping.
